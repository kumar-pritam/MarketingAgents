## Overview

GEO Agent is an AI-powered brand visibility auditor that measures how a brand appears — or fails to appear — in responses generated by large language models (LLMs). It quantifies **AI Share of Voice (AI SoV)** against competitors, identifies topic-level citation gaps, and generates a prioritized content action plan to improve LLM discoverability.

As buyers increasingly use ChatGPT, Perplexity, Claude, and Gemini as their primary research tools, traditional SEO metrics are no longer sufficient. GEO Agent provides the missing signal: **how visible is your brand when an AI answers buyer questions?**

---

## Problem Statement

### What's Happening
Marketing and brand teams invest heavily in SEO, paid media, and content — but have zero visibility into how LLMs respond when a potential buyer asks:
- *"What is the best cyber insurance for SMEs?"*
- *"Which CRM is most trusted by mid-market companies?"*
- *"Compare Chubb vs AXA for business insurance."*

LLM responses are increasingly the first touchpoint in a buyer's research journey. Yet no tool gives brands a structured view of their AI presence.

### The Gap
- Traditional SEO tools (Semrush, Ahrefs) track Google rankings — not LLM citations.
- Brand monitoring tools (Mention, Brandwatch) track social and news — not AI responses.
- No tool today offers continuous, structured AI Share of Voice measurement.

### The Consequence
Brands are invisible in the channel that is growing fastest, with no data to act on.

## Functional Requirements

### FR-01: Brand & Competitor Setup
- User inputs brand name, product category, industry vertical
- User inputs up to 5 competitor brand names
- User inputs up to 10 target keywords / topics
- System validates inputs and generates a brand context object
- **Optional:** Connect Google Search Console for owned content enrichment

### FR-02: Query Generation Engine
- System auto-generates 20–30 queries simulating real buyer search behavior
- Query types must include:
  - `Best [category] for [use case]` — informational
  - `Compare [brand] vs [competitor]` — comparative
  - `Which [category] is most trusted for [topic]` — evaluative
  - `Top [category] providers in [region/industry]` — ranking
  - `What do experts recommend for [topic]` — authority-seeking
- Queries must be seeded with brand context (category, competitors, keywords)
- User can review and edit generated queries before running

### FR-03: LLM Query Runner
- Each query is sent to OpenRouter API using the `meta-llama/llama-3.3-70b-instruct:free` model
- OpenRouter API is OpenAI-SDK compatible — use `openai` Python package with a custom `base_url`
- System prompt frames the model as a neutral buyer/researcher, not an assistant
- Raw LLM responses are stored per query with timestamp
- Runner must handle API rate limits gracefully with retry logic (3 attempts, exponential backoff)
- Runner logs remaining daily quota after each call; warns user at 150/200 requests used
- **Free tier capacity:** 200 requests/day → maximum 8 full audits (25 queries each) per day
- Estimated token cost: **$0.00** on free tier

### FR-04: Response Parser
For each LLM response, the parser must extract:
- **Brand mentioned?** (Yes / No)
- **Competitor mentioned?** (list each competitor found)
- **Brand position** (1st mention / 2nd / buried / not present)
- **Sentiment** (Positive / Neutral / Negative / Mixed)
- **Context of mention** (recommended, compared, mentioned in passing, criticized)
- **Source signal** (does the response cite a URL, report, or reference?)

### FR-05: AI Share of Voice Scoring
Compute the following scores from parsed responses:

```
AI Share of Voice (%) = (Queries where brand is mentioned / Total queries) × 100

Visibility Score (0–100) = weighted sum of:
  - Mention rate: 40%
  - Average position score: 30% (1st mention = 10pts, 2nd = 7pts, buried = 3pts, absent = 0)
  - Sentiment score: 30% (Positive = 10, Neutral = 6, Negative = 2, Mixed = 4)

Gap Score per topic = Topics where competitor appears AND brand does not
```

Competitor scores computed using the same formula for benchmarking.

### FR-06: Gap Detection
- System identifies topics / query clusters where brand is absent
- For each gap, system identifies which competitors dominate
- Gaps are ranked by: frequency of competitor mention × query search intent weight
- Output: Top 5 priority gaps with "who owns this topic" context

### FR-07: Action Plan Generator
- LLM (via OpenRouter) generates a structured content action plan from scoring data
- Each recommendation must include:
  - **Topic to target** (specific, not generic)
  - **Content format** (blog, comparison page, thought leadership, FAQ)
  - **Why this works** (which query type / buyer intent it addresses)
  - **Urgency level** (Act Now / Watch / Low Priority)
  - **Competitor to displace** (named)
- Minimum 5 recommendations, maximum 10
- Written in plain English suitable for sharing with a marketing team

### FR-08: Report Output
- Full audit report displayed in the UI
- Exportable as PDF (Phase 2) and Google Sheets (Phase 2)
- Phase 1: Copy-paste ready text report with scores and action plan
- Report header includes: brand name, audit date, total queries run, runtime

### FR-09: Audit History (Phase 2)
- Each audit result stored locally as JSON
- History tab shows past audits with trend sparklines
- Week-over-week AI SoV delta shown per brand

### Agent Personas

**Query Builder** — Operates as a category researcher. Given brand context, thinks like a buyer at different stages of the purchase funnel (awareness → evaluation → decision) and constructs queries accordingly.

**Runner Agent** — Operates as a neutral buyer with no brand preference. System prompt explicitly instructs the model to respond as a user researching options, not as an assistant helping a brand.

**Parser Agent** — Operates as a structured data extractor. Given raw LLM response text, extracts a JSON object of mentions, positions, and sentiments. Deterministic output format enforced.

**Report Generator** — Operates as a senior brand strategist. Given the scoring data and gap analysis, writes an executive-quality content action plan with specific, named recommendations.

## Integrations

All integrations beyond OpenRouter are **optional**. The core agent runs fully standalone with only the OpenRouter API key.

### Integration 1: OpenRouter API *(Required)*
- **Purpose:** LLM querying (Runner Agent) and action plan generation (Report Generator)
- **Auth:** API key via `.env` file (`OPENROUTER_API_KEY`)
- **Base URL:** `https://openrouter.ai/api/v1` (OpenAI-compatible — use the `openai` Python SDK, just swap the base URL)
- **Primary Model:** `meta-llama/llama-3.3-70b-instruct:free` — GPT-4 level quality, strong instruction following, ideal for structured extraction tasks
- **Fallback Model:** `openrouter/free` — OpenRouter's smart auto-router; selects best available free model per request automatically
- **Free tier:** Fully free. No credit card required. Rate limits: 20 requests/minute, 200 requests/day (resets daily)
- **⚠️ Capacity constraint:** At 25 queries per audit, the free tier supports a maximum of **8 audits/day**. Sufficient for prototyping and WTP validation. Upgrade to a paid OpenRouter plan (~$5 credit) to remove limits for demos.
- **Setup:**
  1. Create account at openrouter.ai (no credit card)
  2. Go to Keys → Create API Key
  3. Copy key → paste as `OPENROUTER_API_KEY` in `.env`
- **API call pattern:**
```python
from openai import OpenAI

client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key=os.getenv("OPENROUTER_API_KEY")
)

response = client.chat.completions.create(
    model="meta-llama/llama-3.3-70b-instruct:free",
    messages=[
        {"role": "system", "content": "You are a buyer researching solutions..."},
        {"role": "user", "content": query}
    ]
)
```

### Integration 2: Google Search Console API *(Optional)*
- **Purpose:** Pull owned content performance data to enrich gap analysis (which pages are already ranking? Which queries drive clicks?)
- **Auth:** OAuth 2.0 with Google account
- **Free tier:** Fully free, your property only
- **Scopes required:** `https://www.googleapis.com/auth/webmasters.readonly`
- **Setup flow:**
  1. Create project at console.cloud.google.com
  2. Enable Search Console API
  3. Create OAuth 2.0 credentials (Desktop App)
  4. Download `credentials.json` → place in `/integrations/`
  5. On first run, browser opens for Google login → token saved to `token.json`
- **Graceful fallback:** If not connected, audit runs without owned content layer. UI shows "Connect GSC for deeper gap analysis" prompt.

### Integration 3: Google Sheets API *(Optional)*
- **Purpose:** Log audit results over time for trend tracking; share reports with stakeholders
- **Auth:** Same Google OAuth credentials as GSC (reuse token)
- **Free tier:** Fully free
- **Scopes required:** `https://www.googleapis.com/auth/spreadsheets`
- **Setup flow:** Add Sheets scope to same OAuth credentials; no separate setup needed
- **Graceful fallback:** Results saved to local JSON only

### Integration 4: Meta Ad Library API *(Optional — Phase 2)*
- **Purpose:** Cross-reference competitor ad messaging with LLM citation topics to detect strategy signals
- **Auth:** Facebook Developer account + App Token (free)
- **Free tier:** Fully free, read-only
- **Setup flow:**
  1. Create app at developers.facebook.com
  2. Add Marketing API product
  3. Generate App Access Token
  4. Store as `META_AD_LIBRARY_TOKEN` in `.env`
- **Graceful fallback:** Excluded from analysis if token absent
